# CRUDExplorer 移行ギャップ分析

移行元（VB.NET WinForms frmAnalyzeQuery）と移行後（C# Avalonia AnalyzeQueryWindow）の  
クエリ解析画面を中心に、不足している機能・表示内容を詳細に整理する。

> **備考**: スクリーンショットにはソースコード（original/CRUDExplorer2011）に存在しない  
> UI要素（タブコントロール、整形ボタン、自動解析チェックボックス、行番号表示）が確認される。  
> これらは実行バイナリ側で後から追加された機能と推定されるため、本文書ではソースコードと  
> スクリーンショットの両方から機能を網羅する。

## 10. ガイドバー（txtGuide）— 完全欠落

- カーソル位置のワードがSELECT句の別名の場合 → `objQuery.ExpandSelect()` で対応する式を展開表示

### 11.4 ワード取得メソッド

オリジナルには3つの異なるワード取得ロジックがある：

| メソッド | 区切り文字 | 用途 |
|---------|-----------|------|
| `GetCursorTableName()` | スペース,カンマ,括弧,演算子,改行 | テーブル名取得（ドットの左側） |
| `GetCursorTableColumnName()` | 同上 | テーブル.カラム名取得（選択状態にする） |
| `GetCursorWord()` | 同上+ドット | 単一ワード取得（選択範囲があればそれを使用） |

移行版では `OnSqlEditorDoubleTapped` で `char.IsLetterOrDigit || '_'` の範囲選択のみ。  
**`%` を含むサブクエリ参照や `.` を含むテーブル.カラム参照の認識ができない。**
